# Glossary

Domain-specific terms used across the cps-assurance repository.

Evidence provenance follows the repository convention: `[F,n]` verified fact, `[S,n]` heuristic, `[I,n]` inference, with confidence levels {50,70,80,90}.

---

| Abbreviation | Full name | Definition | Standard / source reference |
|-------------|-----------|------------|----------------------------|
| ATEX | ATmospheres EXplosibles | EU Directive 2014/34/EU governing equipment and protective systems intended for use in potentially explosive atmospheres. | Directive 2014/34/EU [F,90] |
| BPCS | Basic Process Control System | The control system that manages a process under normal operating conditions; distinct from the SIS which handles safety functions. | IEC 61511-1 [F,90] |
| CAE | Claims, Arguments, Evidence | A structured assurance case notation for building safety and security arguments linking top-level claims to supporting evidence through explicit argument steps. | ISO/IEC 15026-2 [S,85] |
| CCF | Common Cause Failure | A failure of multiple components due to a single shared cause (e.g., environmental stress, design flaw, or supply chain defect), defeating redundancy. | IEC 61508-6 Annex D [F,90] |
| CE | Conformite Europeenne | Marking affixed to products indicating conformity with all applicable EU directives and regulations; required before placing a product on the EU market. | Regulation (EC) No 765/2008 [F,90] |
| CO (Conduit) | Conduit | A logical grouping of communication channels connecting two zones in the IEC 62443 zone/conduit model, with defined protocols, direction, protection, and monitoring. | IEC 62443-3-2 [F,90] |
| CPS | Cyber-Physical System | A system where computational processes interact with physical processes through sensors, actuators, and control loops, creating tight coupling between cyber and physical domains. | General term; NIST definition [S,85] |
| CRA | Cyber Resilience Act | EU Regulation 2024/2847 establishing cybersecurity requirements for products with digital elements placed on the EU market, including SBOM, vulnerability handling, and ENISA reporting. | Regulation (EU) 2024/2847 [F,90] |
| CSIRT | Computer Security Incident Response Team | A team responsible for receiving, analysing, and responding to cybersecurity incidents; CRA requires manufacturer coordination with relevant CSIRTs. | CRA Art. 14; NIS2 Art. 12 [F,90] |
| CVD | Coordinated Vulnerability Disclosure | A structured process for reporting, triaging, and publishing information about vulnerabilities in a coordinated manner between finder, vendor, and affected parties. | CRA Annex I Part II; ISO/IEC 29147 [F,90] |
| CVE | Common Vulnerabilities and Exposures | A publicly catalogued identifier (CVE-YYYY-NNNNN) for a known cybersecurity vulnerability, maintained by the CVE Program. | MITRE CVE Program [F,90] |
| CVSS | Common Vulnerability Scoring System | A standardised framework for rating the severity of software and hardware vulnerabilities on a 0.0-10.0 scale. | FIRST CVSS v3.1 / v4.0 [S,85] |
| DC (Diagnostic Coverage) | Diagnostic Coverage | The fraction of dangerous failures that are detected by online diagnostics, expressed as a percentage; categorised as None (<60%), Low (60-<90%), Medium (90-<99%), High (>=99%). | IEC 61508-2 [F,90] |
| DPI | Deep Packet Inspection | Network monitoring technique that examines the payload of packets (not just headers) to detect anomalous or malicious content in OT protocols such as Modbus or EtherNet/IP. | General networking term; used in IEC 62443-3-3 context [S,80] |
| EHSR | Essential Health and Safety Requirement | A mandatory requirement defined in Annex III of the Machinery Regulation 2023/1230 that machinery must satisfy to be placed on the EU market. | Machinery Regulation 2023/1230 Annex III [F,90] |
| EMC | Electromagnetic Compatibility | EU Directive 2014/30/EU requiring that electrical/electronic equipment neither generates excessive electromagnetic interference nor is unduly affected by it. | Directive 2014/30/EU [F,90] |
| ENISA | European Union Agency for Cybersecurity | The EU agency responsible for cybersecurity; under CRA, manufacturers must report actively exploited vulnerabilities to ENISA. | Regulation (EU) 2019/881; CRA Art. 14 [F,90] |
| FMEDA | Failure Modes, Effects, and Diagnostic Analysis | A systematic hardware reliability analysis method that classifies each component failure mode as safe/dangerous and detected/undetected to calculate PFD, PFH, SFF, and DC. | IEC 61508-2; IEC 62380 [F,90] |
| FR (Foundational Requirement) | Foundational Requirement | One of seven security requirement categories in IEC 62443 (FR 1-7): identification/authentication, use control, system integrity, data confidentiality, restricted data flow, timely response, resource availability. | IEC 62443-3-3 [F,90] |
| GSN | Goal Structuring Notation | A graphical notation for structured assurance cases, representing goals, strategies, contexts, assumptions, justifications, and evidence as linked nodes. | GSN Community Standard v3 [S,85] |
| HARA | Hazard Analysis and Risk Assessment | The process of identifying hazards, assessing their risk (severity, exposure, avoidability), and determining the required safety integrity level or performance level. | IEC 61508-5; ISO 12100 [F,90] |
| hEN | Harmonised European Standard | A European standard (EN) referenced in the Official Journal of the EU, whose application creates a presumption of conformity with the essential requirements of the referencing directive or regulation. | EU Blue Guide [F,90] |
| HFT | Hardware Fault Tolerance | The number of hardware faults a safety function can tolerate before losing its safety function; determines architecture requirements (1oo1 = HFT 0, 1oo2 = HFT 1, 2oo3 = HFT 1). | IEC 61508-2 Table 2/3 [F,90] |
| HMI | Human-Machine Interface | The operator interface (screen, panel, controls) through which humans interact with the control system; attack target in OT security and human factors concern in functional safety. | General term; IEC 62443-3-3; ISO 13849-1 [S,85] |
| ICS | Industrial Control System | A general term for control systems used in industrial environments, encompassing SCADA, DCS, PLC-based systems, and SIS. | NIST SP 800-82 [S,85] |
| IDS | Intrusion Detection System | A system that monitors network traffic or host activity for signs of malicious behaviour; in OT, typically deployed passively on span/mirror ports to avoid disrupting control traffic. | General term; IEC 62443-3-3 SR 6.2 [S,80] |
| JSSR | Joint Safety-Security Requirement | A requirement derived from safety-security interaction analysis (SSI) that addresses both safety and security concerns simultaneously, with dual rationale and joint verification. | IEC 63069 concept; cps-assurance convention [S,80] |
| LOPA | Layer of Protection Analysis | A semi-quantitative risk assessment method that evaluates independent protection layers (IPLs) against each hazard scenario to determine whether the risk is tolerable or additional SIL credit is needed. | IEC 61511-3 [F,90] |
| LVD | Low Voltage Directive | EU Directive 2014/35/EU covering the safety of electrical equipment operating between 50-1000V AC or 75-1500V DC. | Directive 2014/35/EU [F,90] |
| MFA | Multi-Factor Authentication | Authentication requiring two or more independent factors (knowledge, possession, inherence); required for engineering workstation and remote access to OT systems. | IEC 62443-3-3 SR 1.1; NIS2 Art. 21 [S,85] |
| NB (Notified Body) | Notified Body | An organisation designated by an EU member state to assess conformity of products with the requirements of a directive/regulation before they are placed on the market. | EU Decision 768/2008/EC [F,90] |
| NIS2 | Network and Information Security Directive 2 | EU Directive 2022/2555 establishing cybersecurity obligations for essential and important entities, including incident reporting, risk management, and supply chain security. | Directive (EU) 2022/2555 [F,90] |
| OT | Operational Technology | Hardware and software that detects or causes changes through direct monitoring and control of physical devices, processes, and events in industrial environments. | Gartner definition; IEC 62443 context [S,85] |
| PED | Pressure Equipment Directive | EU Directive 2014/68/EU governing the design, manufacture, and conformity assessment of pressure equipment and assemblies with a maximum allowable pressure greater than 0.5 bar. | Directive 2014/68/EU [F,90] |
| PFD | Probability of Failure on Demand | The probability that a safety function will fail to perform its intended function when demanded; the primary reliability metric for low-demand mode safety functions. | IEC 61508-4 [F,90] |
| PFH | Probability of Dangerous Failure per Hour | The average frequency of dangerous failure per hour; the primary reliability metric for high-demand or continuous-mode safety functions. | IEC 61508-4 [F,90] |
| PIAL | Physics-Informed Abstraction Layer | A boundary enforcement model that imposes runtime contracts at every interface where cyber and physical domains interact, covering: process isolation (safety function independence from BPCS); sensor ingestion (input range, rate, and freshness invariants); actuator command (authority limits and rate-of-change clamping); and logic output (value bounds and timing deadline). Applied across all Purdue-level trust boundaries and within safety function interfaces. | cps-assurance convention [S,80] |
| PL (Performance Level) | Performance Level | A discrete level (a through e) used to specify the required ability of a safety-related control system to perform a safety function under foreseeable conditions, per ISO 13849-1. | ISO 13849-1:2023 [F,90] |
| PLC | Programmable Logic Controller | An industrial digital computer adapted for control of manufacturing processes, I/O, and automation; may host safety functions (safety PLC) or basic process control. | General term; IEC 61131 [F,90] |
| PST (Process Safety Time) | Process Safety Time | The time between a hazardous event occurring and the safety function achieving the safe state; drives timing budgets for sensor ingestion, compute, and actuator response. | IEC 61508-4; IEC 61511-1 [F,90] |
| RED | Radio Equipment Directive | EU Directive 2014/53/EU covering radio equipment placed on the EU market, including cybersecurity requirements activated by Delegated Regulation (EU) 2022/30. | Directive 2014/53/EU [F,90] |
| SBOM | Software Bill of Materials | A machine-readable inventory of software components, libraries, and dependencies in a product; required by CRA for products with digital elements. | CRA Annex I; NTIA SBOM [F,90] |
| SFF | Safe Failure Fraction | The fraction of the overall failure rate of an element that results in either a safe failure or a detected dangerous failure; used with HFT to determine architectural constraints. | IEC 61508-2 [F,90] |
| SIL | Safety Integrity Level | A discrete level (1 through 4, with 4 being the highest) that specifies the required probability of a safety function performing satisfactorily under all stated conditions within a stated period of time. | IEC 61508-1 [F,90] |
| SIS | Safety Instrumented System | A system composed of sensors, logic solvers, and final elements designed to bring a process to a safe state when predetermined conditions are violated; must be independent from the BPCS. | IEC 61511-1 [F,90] |
| SL-A | Security Level — Achieved | The actual security level measured or assessed for a zone or component, based on the controls currently in place; compared against SL-T to determine gaps. | IEC 62443-3-3 [F,90] |
| SL-C | Security Level — Capability | The security level that a component or system is capable of providing based on its design and implementation, independent of deployment configuration. | IEC 62443-4-2 [F,90] |
| SL-T | Security Level — Target | The desired security level assigned to a zone based on risk assessment; drives the selection of security controls per IEC 62443-3-3 foundational requirements. | IEC 62443-3-2 [F,90] |
| SPDX | Software Package Data Exchange | An open standard (ISO/IEC 5962:2021) for communicating software bill of materials information, including component provenance, licensing, and security references. | ISO/IEC 5962:2021 [F,90] |
| SSI | Safety-Security Interaction | An identified interaction between safety and security domains classified by type (SS-1 through SS-7), requiring joint analysis and resolution to prevent one domain from undermining the other. | IEC 63069; cps-assurance convention [S,80] |
| STPA | System-Theoretic Process Analysis | A hazard analysis technique based on systems theory (STAMP) that identifies unsafe control actions and causal scenarios by analysing control structures rather than component failures. | MIT/Leveson; STPA Handbook [S,85] |
| STRIDE | Spoofing, Tampering, Repudiation, Information disclosure, Denial of service, Elevation of privilege | A threat classification model that categorises security threats into six types; adapted in this repository for OT with physical consequence tracing. | Microsoft SDL [S,85] |
| UCA | Unsafe Control Action | A control action that, in a particular context and worst-case environment, leads to a hazard; identified during STPA analysis across four types (not provided, provided incorrectly, wrong timing, applied too long/stopped too soon). | STPA Handbook [S,85] |
| WORM | Write Once Read Many | A storage policy ensuring that evidence artifacts, once written, cannot be modified or deleted; used in the evidence pipeline for audit integrity. | General term; platform-assurance convention |
| Zone | Zone (IEC 62443) | A logical or physical grouping of assets that share common security requirements; each zone is assigned a Security Level Target (SL-T) and protected by conduits at its boundaries. | IEC 62443-3-2 [F,90] |
