# Annual CPS Assurance Review Checklist

**Year:** YYYY
**Review lead:** Safety Manager + OT Security Lead
**Date completed:** YYYY-MM-DD

## Safety

- [ ] Hazard register reviewed; all H-n entries current
- [ ] SIL/PL allocations still valid (no system changes invalidating them)
- [ ] Proof tests completed per schedule; results documented
- [ ] Safety function test records reviewed; no outstanding failures
- [ ] Operator competency validated (training records current)
- [ ] Any safety incidents reviewed; CAPA complete or on-track

## OT security

- [ ] Zone/conduit model reviewed; matches current OT network topology
- [ ] SL-T vs SL-A gap assessment updated
- [ ] PLC/SIS firmware versions inventoried; known vulnerabilities checked
- [ ] Default credential audit completed (no defaults remaining)
- [ ] OT monitoring effectiveness reviewed (false positive rate, detection coverage)
- [ ] Vendor remote access audit completed (sessions reviewed, unused accounts removed)
- [ ] OT incident response procedure tested (tabletop or live exercise)

## Safety-security interaction

- [ ] SSI register reviewed; all entries current
- [ ] Joint requirements (JSSR) verification status updated
- [ ] Any new system changes assessed for safety-security interaction
- [ ] Conflict resolution log reviewed; no stale unresolved conflicts

## Product conformity

- [ ] Product register reviewed; all PROD-n entries current
- [ ] SBOMs regenerated for products with component changes
- [ ] Vulnerability disclosure policy still published and reachable
- [ ] CRA ENISA reporting readiness verified (process test)
- [ ] CE marking still valid (no changes invalidating conformity)
- [ ] Technical documentation updated where system changes occurred
- [ ] Support period and update commitments still on track

## Regulatory

- [ ] Harmonised standards publications checked (any new hEN relevant to products?)
- [ ] Regulatory changes checked (CRA implementing acts, Machinery Reg transition, RED delegated act)
- [ ] NIS2 compliance reviewed (if operator is essential/important entity)

## Evidence

- [ ] All review records stored in evidence pipeline
- [ ] This checklist signed and stored
